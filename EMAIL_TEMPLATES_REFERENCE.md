# Email Templates - Quick Reference Guide

A quick reference for all automated email notifications sent by Zehlora.

---

## 📧 Email Templates Overview

| Template | Trigger | Recipient | Key Info |
|----------|---------|-----------|----------|
| **Invitation** | Admin creates invitation | New team member | Invitation link (7 days), org name, role |
| **Member Added** | Admin creates user directly | New team member | Temp username & password, login URL |
| **Welcome** | User accepts invitation | New team member | Dashboard link, feature overview, next steps |
| **Password Changed** | User changes password | Team member | Timestamp, security verification |

---

## 🎯 Detailed Templates

### 1. Admin Role Invitation Email

**When sent:**
- Admin clicks "Invite Team Member" and enters email
- Admin shares the link with the person
- Email arrives automatically

**Who receives it:**
- The email address provided in the invitation form

**What's in the email:**
```
Subject: You're invited to join [Organization] on Zehlora

Header: Welcome to Zehlora! / You've been invited to join the team

Body:
- Greeting with recipient's name
- Organization name
- Role being invited to
- Big "Accept Invitation & Set Password" button
- Expiry info (7 days)
- Link backup (copy/paste)
```

**Design:**
- Gold/amber accent color
- Professional business styling
- Responsive (works on mobile & desktop)
- Branded with Zehlora logo in footer

**Expires:**
- 7 days from creation
- Admin can revoke early
- Admin can re-invite (creates new token)

---

### 2. Member Added with Credentials Email

**When sent:**
- Admin uses "Add Staff Member" form
- Password is auto-generated by system
- Email sent automatically with credentials

**Who receives it:**
- The email address of the new staff member

**What's in the email:**
```
Subject: Your [Role] account is ready - Zehlora

Header: Welcome to the Team! / Your account has been created

Body:
- Greeting with recipient's name
- Role information
- **EMAIL ADDRESS (username)** - highlighted in green box
- **TEMPORARY PASSWORD** - highlighted in green box
- Big "Login to Zehlora" button
- ⚠️ Security warning box with 4 steps:
  1. Login with credentials
  2. Change password immediately
  3. Set up two-factor authentication
  4. Never share password
- Next steps (complete profile, etc.)
```

**Design:**
- Green accent color (success state)
- Credentials clearly visible in colored box
- Security warnings prominent
- Clear call-to-action

**Password:**
- Auto-generated if not provided
- 12 characters, random
- Only shown once to admin & in this email
- Cannot be recovered (must reset)

---

### 3. Welcome Email (After Successful Signup)

**When sent:**
- User clicks invitation link
- User sets their own password
- Account is created successfully
- Email sent automatically

**Who receives it:**
- The invited user's email address

**What's in the email:**
```
Subject: Welcome to Zehlora - [Organization]

Header: 🎉 Welcome to Zehlora! / Your account is all set up and ready to go

Body:
- Congratulations message
- Organization name
- Big "Go to Dashboard" button
- Feature highlights (with icons):
  - 📊 Inventory
  - 💰 Finance
  - 🏪 POS
  - 🌐 E-Commerce
- Getting started steps:
  1. Complete profile
  2. Set up shops/locations
  3. Import inventory
  4. Explore features
```

**Design:**
- Cyan/turquoise accent color
- Feature cards with icons
- Friendly, welcoming tone
- Encouraging next steps

**Timing:**
- Sent immediately after password set
- No delay

---

### 4. Password Changed Notification Email

**When sent:**
- User changes their password in Settings
- Sent as security notification

**Who receives it:**
- The user who changed the password

**What's in the email:**
```
Subject: Password Changed

Header: Password Changed / Security notification

Body:
- Confirmation: "Your password was successfully changed at [TIME]"
- Info box with two scenarios:
  * "If this was you: You can ignore this email"
  * "If this wasn't you: Your account may be compromised - reset password & contact support"
- Security tips:
  - Use strong, unique password
  - Never share password
  - Enable two-factor authentication
  - Log out from unused devices
```

**Design:**
- Purple accent color
- Security-focused messaging
- Clear action items if account compromised

## 🔧 How These Emails Are Used

### Scenario 1: Inviting a New Team Manager

**Steps:**
1. Admin goes to Settings > Team
2. Clicks "Invite Team Member"
3. Fills in:
   - Email: `manager@company.com`
   - Name: `John Manager`
   - Role: `Manager`
4. Clicks "Send Invitation"

**What happens:**
- ✅ Invitation is created in system
- ✅ Email sent to `manager@company.com` with invitation link
- ✅ Admin sees a success message with the token
- ✅ Admin can copy the link to share manually if needed

**Manager receives:**
- Email with "Accept Invitation & Set Password" button
- Manager clicks button → fills in password → account created
- Manager receives second email: "Welcome to Zehlora!"

---

### Scenario 2: Adding a POS Staff Member

**Steps:**
1. Admin goes to Settings > Team
2. Clicks "Add Staff Member"
3. Fills in:
   - Name: `Ram`
   - Email: `ram@shop.com`
   - Role: `POS User`
   - Shop: `Main Store`
4. Clicks "Create Account"

**What happens:**
- ✅ Account created with auto-generated password
- ✅ Password shown to admin (one-time)
- ✅ Email sent to `ram@shop.com` with credentials
- ✅ Admin writes password on paper/shares securely

**Ram receives:**
- Email with username and temporary password
- Instructions to login and change password
- Email emphasizes security steps

**Ram's first login:**
- Uses email + temp password
- System forces password change
- Ram sets new password
- Ram is in!

## 📋 Email Checklist for Admins

### Before Inviting Someone
- [ ] Verify email address is correct
- [ ] Confirm their role/permissions needed
- [ ] Check they're not already in the system

### After Inviting
- [ ] Verify they received the email
- [ ] Share the link securely (WhatsApp, etc.)
- [ ] Set a reminder if they don't accept in 3-4 days
- [ ] Can revoke and re-invite if needed

### Before Adding Staff Member
- [ ] Have correct email address
- [ ] Know which shop they'll work at
- [ ] Confirm their role

### After Adding Staff Member
- [ ] Share credentials securely
- [ ] Remind them to change password
- [ ] Confirm they can login
- [ ] Do NOT reuse generated passwords

## 🚨 What if Email Doesn't Work?

### Signs Email System is Down
- No emails arriving after creating invitation
- Server logs show "✗ Failed to send email"
- Server shows "SMTP not configured" on startup

### What to Do
1. Check with IT / DevOps
2. Verify SMTP credentials haven't changed
3. Check Hostinger account isn't suspended
4. Try manually sending test email from Hostinger Webmail

### Workaround While Fixing
- Share invitation link manually (admin gets it)
- Share temporary credentials manually
- Update credentials in `.env` when fixed
- Restart server

## 💡 Tips & Tricks

### Best Practices
1. **Always verify email** before inviting someone
2. **Revoke old invitations** if you re-invite someone
3. **Don't reuse passwords** across staff
4. **Check spam folders** if someone says they didn't receive email
5. **Share credentials securely** (WhatsApp, not email)

### Common Questions

**Q: Can I edit the email templates?**
A: Templates are built-in. For changes, contact IT/DevOps.

**Q: Can I send custom emails?**
A: Only the 4 templates above are automated.

**Q: What if they don't respond?**
A: Revoke the invitation and re-invite after a few days.

**Q: Can I see who received emails?**
A: Check invitation list in Settings > Team.

**Q: How long before the link expires?**
A: 7 days for invitations. Account is active immediately.
